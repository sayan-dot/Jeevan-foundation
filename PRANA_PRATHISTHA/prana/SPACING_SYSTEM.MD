# Spacing System

**Part of:** Prana — The Design Language of Jeevan

**Version:** 1.0

**Status:** Draft

---

# Purpose

The spacing system defines how elements are positioned relative to one another throughout Jeevan.

Consistent spacing creates rhythm, improves readability, and reduces visual clutter.

Spacing should help users understand relationships between elements without requiring conscious thought.

---

# Philosophy

Spacing is not empty space.

Spacing is communication.

The distance between elements tells users what belongs together and what should be perceived separately.

Well-designed spacing creates calm interfaces.

Poor spacing creates confusion.

---

# Principles

- Consistency creates familiarity.
- Whitespace improves comprehension.
- Related elements stay close.
- Unrelated elements stay apart.
- Never reduce spacing simply to fit more content.

---

# Spacing Scale

Prana uses a consistent spacing scale based on an 8-point grid.

The exact spacing tokens will be defined during implementation.

Example:

- XS
- SM
- MD
- LG
- XL
- 2XL

This scale should be used consistently throughout the application.

---

# Layout Rhythm

Every screen should maintain a predictable visual rhythm.

Spacing should help users naturally scan information from top to bottom.

Avoid inconsistent gaps between similar elements.

---

# Component Spacing

Every component should define:

- Internal Padding
- External Margin
- Gap Between Elements

These values should follow the global spacing scale.

---

# Screen Padding

All screens should use consistent horizontal and vertical padding.

Padding should create a comfortable reading area while maximizing usability.

---

# Accessibility

Adequate spacing improves accessibility by:

- Reducing accidental taps
- Improving readability
- Increasing touch accuracy
- Reducing visual fatigue

Spacing should never be sacrificed to fit additional content.
