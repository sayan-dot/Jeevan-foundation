# Purpose

This document defines the primary users of Jeevan.

The purpose of these personas is to ensure every product decision is made for real people instead of generic "users."

Every feature should improve the experience for at least one persona.

## About Ramesh

Ramesh owns an Android phone that his son purchased for him.

He mostly uses:

- WhatsApp
- YouTube
- Camera

He rarely installs new applications because he worries about making mistakes.

When applications become confusing, he usually asks his son for help.

---

## Goals

- Never miss medicines
- Stay healthy
- Use the phone independently
- Feel confident using the application

---

---

## Design Requirements

# Design Principles Derived From Personas

These personas lead to several product decisions.

- Large touch targets
- Minimal reading
- Icons before text
- Calm interface
- Friendly illustrations
- Regional languages
- Accessibility by default
- Voice assistance
- Guided interactions

---

# Validation

Whenever a new feature is proposed, ask:

Would Ramesh understand this?

Would Sunita find this useful?

Would Anita be able to use this without help?

If the answer is "No," the feature should be redesigned.
