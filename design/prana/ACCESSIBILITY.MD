# Accessibility Guidelines

**Version:** 1.0  
**Last Updated:** 30 June 2026  
**Status:** Active

---

# Introduction

Accessibility is not a feature added after development.

It is a fundamental design requirement.

Every person deserves equal access to healthcare, regardless of age, language, physical ability, or technical experience.

At Jeevan, accessibility is considered from the first sketch, not the final release.

---

# Our Philosophy

We believe good accessibility creates better products for everyone.

Designing for elderly users naturally produces interfaces that are easier, clearer, and more enjoyable for all users.

Accessibility is not about designing for limitations.

It is about removing unnecessary barriers.

---

# Accessibility Principles

## Design for Independence

Users should be able to complete important healthcare tasks without assistance whenever possible.

The interface should encourage confidence rather than dependence.

---

## Reduce Cognitive Load

Healthcare already requires users to remember medicines, appointments, and schedules.

The interface should never add unnecessary mental effort.

Reduce:

- Choices
- Steps
- Reading
- Memory requirements

Whenever possible.

---

## Prioritize Recognition Over Recall

Users should recognize actions instead of remembering them.

Examples include:

- Familiar icons
- Consistent layouts
- Predictable navigation
- Visual medicine cards

The interface should always answer:

"What can I do next?"

without requiring thought.

---

## Design for Imperfect Conditions

Users may:

- Have poor eyesight
- Use the app outdoors
- Be distracted
- Be tired
- Feel anxious
- Use one hand
- Wear glasses

The interface should continue working well in these situations.

---

# Typography

Text should remain readable without effort.

Guidelines

- Large default font sizes
- High contrast
- Generous line spacing
- Avoid decorative fonts
- Avoid long paragraphs

Reading should never feel exhausting.

---

# Touch Targets

Interactive elements should be comfortable to tap.

Guidelines

- Large buttons
- Generous spacing
- Avoid tightly packed actions
- Minimize accidental touches

Users should never fear tapping the wrong button.

---

# Color Usage

Color should never be the only way information is communicated.

Every important status should include:

- Icon
- Label
- Color

Example

✓ Taken

instead of

Green only.

This ensures information remains understandable for users with color vision deficiencies.

---

# Motion

Motion should support understanding.

Avoid:

- Flashing animations
- Rapid movement
- Continuous distractions

Use motion only to:

- Guide
- Confirm
- Explain

---

# Language

Language should remain:

- Simple
- Clear
- Respectful

Avoid technical healthcare terminology whenever possible.

Future versions of Jeevan should support regional languages and voice guidance.

---

# Error Prevention

Prevent mistakes before they happen.

Prefer:

Confirmation before destructive actions.

Clear explanations.

Simple recovery.

Healthcare mistakes can have serious consequences.

The interface should help users avoid them.

---

# Feedback

Every important action should provide immediate feedback.

Examples include:

- Visual confirmation
- Gentle animation
- Haptic feedback
- Clear success messages

Users should never wonder whether an action was completed.

---

# Voice Accessibility

Future versions of Jeevan should provide voice-guided interactions through Sarathi.

Voice guidance should:

- Speak naturally
- Use simple language
- Give one instruction at a time
- Wait for user interaction
- Never rush the user

The goal is guidance, not automation.

---

# Accessibility Checklist

Before approving a design, ask:

- Can this be understood without reading large amounts of text?
- Is the next action obvious?
- Are touch targets large enough?
- Does color have an alternative indicator?
- Can an elderly user complete the task independently?
- Does the interface reduce anxiety?
- Does the interface reduce mistakes?
- Does the interface respect the user's attention?

---

# Closing Statement

Accessibility is not about helping a small group of users.

Accessibility is about creating healthcare technology that everyone can trust.

When accessibility improves, everyone benefits.
